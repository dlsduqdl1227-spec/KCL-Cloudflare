# TEST_REPORT_STAGE14

## 수정 목적
Stage13에서 추가된 선수 초기화 / 운영 초기화 / 관리자 추가 기능은 유지하되, 중복된 초기화 버튼과 반복 안내를 정리하고 모바일 화면에서 더 깔끔하게 사용할 수 있도록 UI 구조를 재정리했다.

## 반영 내용
1. 등록·권한 관리 화면 상단 정리
   - 대회 선택 카드에는 대회 선택과 현재 탭 새로고침만 남김.
   - 선수 초기화와 운영계정 초기화는 `데이터 초기화` 접이식 영역으로 통합.
   - 위험 작업은 기본적으로 접힌 상태로 두어 실수 클릭 가능성을 낮춤.

2. 전체 엑셀 등록 영역 분리
   - 기존 선수 탭 안에 있던 전체 등록 기능을 탭 밖의 공통 카드로 이동.
   - 선수 탭은 선수 등록/개별 추가/목록만 담당하도록 정리.
   - 운영 권한 탭은 심사위원·팀장·운영진 등록과 목록만 담당하도록 정리.
   - 관리자 탭은 관리자 추가와 관리자 목록만 담당하도록 정리.

3. 모바일 레이아웃 보정
   - 탭을 3단 세로 버튼이 아닌 가로 스크롤형 pill 버튼으로 변경.
   - 모바일에서 대회 선택, 새로고침, 초기화 버튼이 1열로 자연스럽게 정렬되도록 조정.
   - 엑셀 템플릿 다운로드 버튼은 모바일에서 전체 폭으로 확장.
   - 테이블은 기존처럼 가로 스크롤을 유지해 컬럼 잘림을 방지.

4. 초기화 로직 정리
   - 선수 초기화와 운영계정 초기화의 확인 절차를 공통 함수로 통합.
   - 1차 confirm + 2차 정확한 문구 입력 방식을 유지.
   - 입력 문구가 다르면 초기화가 취소되도록 유지.

## 확인한 안전장치
- 선수 초기화는 선택 대회의 participants 데이터만 삭제한다.
- 운영계정 초기화는 선택 대회의 심사위원·팀장·운영진 권한만 삭제한다.
- 관리자 계정은 운영계정 초기화 대상에서 제외된다.
- 마지막 관리자 계정 삭제 방지 로직은 서버에 유지되어 있다.
- 기존 관리자 로그인 시크릿 코드 검증은 유지되어 있다.

## 정적 점검 결과
- `public/registry/index.html` 내부 스크립트 문법 검사 통과.
- `functions/api/rpc.js` 문법 검사 통과.
- 등록·권한 관리 화면 내 중복 ID 없음.
- `allRegistryFile`, `allMsg`, `comp`, `participantsCsv`, `operatorsCsv`, `operatorTable`, `adminTable` 주요 DOM ID가 각 1개씩만 존재함을 확인.
- 중복 초기화 버튼은 제거되고, 초기화 실행 버튼은 `데이터 초기화` 영역에만 존재함.

## 배포 후 현장 확인 권장 항목
1. 관리자 로그인 후 등록·권한 관리 화면 진입.
2. 모바일 화면에서 탭이 가로로 자연스럽게 이동되는지 확인.
3. 선수 등록 탭에서 선수 목록 조회 및 개별 저장 확인.
4. 운영 권한 탭에서 심사위원·팀장 저장 및 목록 조회 확인.
5. 관리자 탭에서 관리자 저장 및 목록 조회 확인.
6. `데이터 초기화`를 열고 2단계 확인 없이 초기화가 실행되지 않는지 확인.
