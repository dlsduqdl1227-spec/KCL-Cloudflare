# TEST REPORT — Stage56 MOB Score Aggregation Audit Final

## 목적
MOB 점수 집계 구조를 룰북 기준과 실제 저장/순위 집계 로직 기준으로 전면 재검수하고, 발견된 오류 가능성을 보정했다.

## 룰북 기준 재확인
- MOB 예선: 브루잉 커피 시연, 창작 메뉴 생략.
- MOB 결선: 브루잉 커피 + 창작 메뉴 시연.
- 심사위원 구조: 헤드 1명, 테크니컬 1명, 센서리 3명.
- 예선 동점 기준: 센서리 점수 → 테크니컬 점수 → 경기 시간.
- 결선 동점 기준: 센서리 점수 → 테크니컬 점수 → 창작 메뉴 점수 → 경기 시간.
- 시간 감점: 제한시간 초과 1~15초 -6점, 16~30초 -24점, 31~60초 -40점, 1분 초과 자동 실격.

## 발견 및 수정 사항

### 1. 서버 순위 동점 기준용 경기시간 파싱 오류 수정
- UI는 MOB 경기시간을 `10분 08.00초` 형태로 저장한다.
- 기존 서버 순위 로직은 이 값을 숫자만 추출해 `1008.00`초로 해석할 수 있었다.
- 결과적으로 점수 총합은 맞아도, 동점자 경기시간 비교가 실제보다 크게 계산될 수 있었다.
- `분/초` 한글 포맷, `mm:ss`, `h:mm:ss`, `min/sec` 포맷을 모두 정상 초 단위로 파싱하도록 수정했다.

### 2. 시간감점 음수 저장 데이터 호환 보정
- 현재 UI는 시간감점을 `6`, `24`, `40`처럼 양수로 저장한다.
- 이전 데이터나 검수 수정 과정에서 `-6`처럼 음수로 저장된 경우, 서버 집계에서 `점수 - (-6)`으로 처리되어 점수가 증가할 수 있는 위험이 있었다.
- MOB 서버 집계에서 시간감점은 항상 절댓값 양수 감점으로 정규화하도록 수정했다.

### 3. 실격 참가자 점수 보존/표시 오류 방지
- 시간초과 실격 또는 수동 실격이 포함된 참가자는 순위에서 실격 처리되어야 한다.
- 기존 로직은 실격 순위는 아래로 보내지만, 세부 합산 점수가 남아 보일 수 있는 구조였다.
- MOB 집계에서 실격 그룹은 최종 랭킹 점수 0점, 집계 기준 `실격`으로 처리되도록 수정했다.

### 4. 총평가 반영점수 0점 보존 조건 보정
- 기존에는 저장된 `총평가 반영점수`가 0이고 원시 점수 합산값이 있으면, 실격이 아니어도 0점을 우선 보존할 수 있었다.
- 이제 실격은 `실격여부=Y` 조건에서만 0점 보존하고, 실격이 아닌 데이터는 원시 평가항목 기준으로 재계산한다.
- 이전 버전 데이터가 일부 0점으로 잘못 저장되어 있어도, 평가항목이 남아 있으면 정상 복원 계산된다.

### 5. MOB 검수 상세 역할 판정 보강
- 검수 상세에서 `T1`, `TJudge`, `TechnicalJudge`, `S1`, `SJudge`, `SensoryJudge` 등의 역할 표기를 안정적으로 인식하도록 보강했다.
- 테크니컬/센서리 항목이 서로 섞여 보일 가능성을 추가로 줄였다.

## 최종 MOB 집계 공식

### 예선
- 최종점수 = 테크니컬 점수 + 센서리 평균점수 - 시간감점
- 동점 기준 = 센서리 평균점수 → 테크니컬 점수 → 경기시간

### 결선
- 최종점수 = 테크니컬 점수 + 창작음료 테크니컬 점수 + 센서리 평균점수 + 창작메뉴 평균점수 - 시간감점
- 동점 기준 = 센서리 평균점수 → 테크니컬 점수 → 창작메뉴 평균점수 → 경기시간

## 회귀 테스트
- `10분 08.00초` → 608초 정상 파싱 확인
- `10:08` → 608초 정상 파싱 확인
- `시간감점 -6` → 6점 감점으로 보정 확인
- 예선 예시: 테크니컬 9, 센서리 24/25/26, 시간감점 6 → 최종 28점 확인
- 결선 예시: 테크니컬 28, 센서리 23/24/25, 창작 19/20/21, 시간감점 24 → 최종 48점 확인
- 실격 포함 그룹 → 랭킹 점수 0점 및 `실격` 기준 확인
