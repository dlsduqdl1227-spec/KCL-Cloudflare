# KCL 디지털 평가 웹앱 Stage70 전체 대회 점수 기준 최종 점검

## 목적
Stage69까지의 MOB 총점 수정 이후, 2026 KCL 7개 대회(KCR, KBC, KCAC, MOC, KTCC, IKRC, MOB)의 순위 반영 점수가 룰북 기준의 총점/평균/동점 기준과 맞는지 다시 점검하고, 코드상 불일치 가능성을 보정했습니다.

## 공통 반영 원칙
- 순위 반영 상태: `검수완료`, `수정완료`
- 순위 제외 상태: `미검수`, `켈리브레이션`
- 실격: 최종 순위 반영점수 0점 처리
- 시간감점: 가산이 아니라 항상 차감 처리
- `수정완료` 데이터도 순위·디브리핑·최종정리 엑셀에 반영
- 켈리브레이션/헤드 데이터는 규정상 순위 반영 대상이 아닌 대회에서 제외

## 이번 Stage70의 핵심 수정

### 1. 전체 대회별 공식 집계 함수를 명시화
기존에는 일부 대회가 공통 평균 로직으로 떨어질 수 있어 운영자가 총점 기준을 오해할 가능성이 있었습니다. Stage70에서는 `aggregateRankingGroup_`에서 7개 대회를 각각 분기해 룰북 기준을 명시했습니다.

### 2. KBC 시간감점 처리 보강
KBC는 평가표 총합의 평균에서 시간감점을 차감하는 구조이므로, 심사위원별 점수에 같은 시간감점이 중복 반영될 위험을 줄이기 위해 `감점 전 합산 평균 - 시간감점 1회` 구조로 보강했습니다.

### 3. MOC / KTCC 중복 제출 방어
MOC와 KTCC는 팀장/운영자가 한 공식 결과를 제출하는 구조에 가깝기 때문에, 동일 참가자 또는 팀의 중복 제출이 있을 경우 평균을 내지 않고 최신 공식 제출 행만 순위에 반영하도록 보강했습니다.

### 4. MOB 제출 확인창 총점 보정
시간초과 실격 또는 시간감점 상황에서 화면 확인창과 저장 행의 총점이 공식 순위 반영점수와 다르게 보일 가능성을 제거했습니다. 이제 MOB 제출 확인창의 총점도 `총점 합산 - 시간감점` 후의 공식 반영점수와 동일합니다.

### 5. KCR 정규화/공식점수 우선 처리
KCR 룰북상 최종 순위는 정규화된 총점을 기준으로 하므로, 데이터에 `정규화점수`, `Normalized Score`, `공식점수`, `순위반영점수`가 있으면 해당 값을 우선 사용하도록 했습니다. 해당 값이 없을 때만 앱 입력 항목 총점을 기준으로 표시합니다.

## 운영 전 최종 확인 필요 사항
실제 운영 DB와 브라우저에서 다음 흐름은 배포 후 한 번 더 확인이 필요합니다.

1. 각 대회별 테스트 선수 1명 등록
2. 평가 제출
3. 관리자 검수 수정
4. 수정완료 저장
5. 검수완료 저장
6. 실시간 순위 확인
7. 디브리핑 상세 확인
8. 최종정리 엑셀 다운로드 확인
9. 대회별 초기화가 해당 대회 데이터만 초기화하는지 확인

## 결론
Stage70은 현재 코드 기준으로 7개 대회 전체의 공식 점수 산정 방식, 시간감점, 실격, 켈리브레이션 제외, 동점 기준을 룰북 구조에 맞춰 재점검하고 보강한 최종 점검본입니다.
