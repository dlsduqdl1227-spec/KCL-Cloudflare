# KCL Assessment Stage17 Final Tested QA

## 목적
Stage16 최종 구현본을 기준으로 실제 운영 시나리오에 가깝게 관리자, 대회팀장, 심사위원 권한을 나누어 직접 테스트했다.

## 추가 보정
1. 검수/순위 관련 권한 강화
   - 심사위원 계정은 평가 제출만 가능하도록 확인했다.
   - 검수 목록, 검수완료 처리, 순위 조회, 순위 상세 조회는 관리자 또는 해당 대회 팀장 권한이 있어야 접근 가능하도록 보강했다.
   - 최종정리파일 다운로드는 기존과 동일하게 관리자/팀장 권한에서만 가능하다.

2. KCAC 이미지 기록 보강
   - 기존에는 KCAC 예선 패턴평가 이미지 저장을 강제했지만, 결선 창작패턴은 제출 전 이미지 저장 강제가 약했다.
   - Stage17에서는 KCAC 패턴평가 전체, 즉 예선 패턴평가와 결선 창작패턴 모두 이미지 스냅샷 저장 후 제출되도록 보강했다.
   - 센서리용 아트는 맛/질감 평가 대상이므로 이미지 저장을 강제하지 않는다.

3. 시스템 상태 버전명 정리
   - 시스템 상태 표시 버전을 `1.0ver-security-stage17-final-tested`로 정리했다.

## 직접 테스트 시나리오

### 1. 계정/권한
- 관리자 로그인: 성공
- 심사위원 로그인: 성공
- 대회팀장 로그인: 성공
- 심사위원의 검수 목록 접근: 차단 확인
- 대회팀장의 검수 목록/검수완료/순위 조회: 성공
- 관리자의 최종정리파일 데이터 조회: 성공

### 2. 평가 제출 및 중복 제출 방지
- KCR 평가 제출: 성공
- 동일 평가 20초 이내 재제출: 중복 저장 차단 확인
- 저장 데이터에 라운드, 심사위원, 역할, 참가자번호, 총점 기록 확인

### 3. 대회별 점수 계산 및 순위
- KCR: `Flavor + Aftertaste + Acidity + Body + Sweetness×2 + Overall` 계산 확인
- KCR 동점 처리: 총점 동점 시 Sweetness 높은 선수가 우선 순위로 정렬되는 것 확인
- KCAC: 2잔 합산 총점 정상 계산 확인
- MOC: 정답수 + 가산점 계산 및 동점 시 종료시간 짧은 순 정렬 확인
- KTCC: Section1~3 정답수 + Section3 가산점 계산 및 동점 시 종료시간 짧은 순 정렬 확인
- IKRC: Flavor×3, Clean Cup×2, Sweetness×2, Acidity, Mouthfeel×2 합산 확인
- KBC: Presentation, Espresso, Signature, Machine, 시간감점 반영 확인
- MOB: 테크니컬 평균 + 센서리 평균 + 창작메뉴 평균 - 시간감점 방식 확인

### 4. 이미지/서명 기록
- KCAC 패턴평가 이미지 저장 후 제출: 성공
- 검수 목록에서 이미지저장 Y, 이미지개수 표시 확인
- 순위 상세에서 저장 이미지 payload 유지 확인
- MOC 서명 제출: 성공
- 서명 데이터가 `signature_data`에 저장되고 순위 상세에서 유지되는 것 확인
- KTCC 서명 제출: 성공

### 5. 최종정리파일 데이터
- `getFinalReport` 데이터 생성 성공
- 순위 데이터 포함 확인
- 전체 점수 rows 포함 확인
- 검수완료 점수 approvedRows 포함 확인
- rawRows에 payload 요약 포함 확인
- KCAC 이미지 저장 여부와 이미지 개수 반영 확인

## 결론
직접 테스트 기준으로 평가 제출, 저장, 중복 방지, 검수완료, 순위 산출, 이미지/서명 보존, 최종정리파일 데이터 생성 흐름은 정상 동작한다.
실제 Cloudflare 배포 후에는 운영 환경의 D1 연결과 브라우저 다운로드 권한 확인을 위해 테스트 선수 1명으로 `제출 → 검수완료 → 순위 → 최종정리 엑셀 다운로드` 흐름만 최종 리허설하면 된다.
