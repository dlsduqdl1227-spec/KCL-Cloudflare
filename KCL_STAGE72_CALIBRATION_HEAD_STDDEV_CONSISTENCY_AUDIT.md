# KCL Stage72 — 헤드 심사위원별 켈리브레이션 표준편차 일관성 검수

## 요청 내용
- 헤드 심사위원마다 MOB/IKRC 켈리브레이션 표준편차 결과값이 다르다는 의견이 있어 재검수.
- 표준편차·헤드차 판정 기준은 Stage71 기준 유지: 1.00점 이내만 양호, 1.01~2.00 주의, 2.00 초과 점검.

## 확인한 원인
1. **서버 켈리브레이션 조회가 로그인한 헤드의 평가팀명으로 필터링될 수 있었음**
   - `requestedTeam` 값이 `Head A`, `센서리1팀`, `테크니컬팀`처럼 계정별로 다르면 같은 참가자/컵이라도 조회되는 일반 심사위원 행 집합이 달라질 수 있음.
   - 이 경우 표준편차 자체가 헤드마다 달라 보일 수 있음.

2. **헤드 기준 비교값이 현재 로그인한 헤드 기준값을 우선 사용하고 있었음**
   - 표준편차는 일반 심사위원 분포 기준이라 동일해야 하지만, 화면의 `헤드차`는 로그인한 헤드 기준값에 따라 달라질 수 있었음.
   - 현장에서는 표준편차와 헤드차가 같은 박스에 표시되므로 “헤드마다 켈리브레이션 결과가 다르다”는 혼선이 발생할 수 있음.

3. **같은 심사위원의 중복 제출이 표준편차 계산에 포함될 수 있었음**
   - 동일 심사위원이 같은 참가자/컵에 대해 다시 제출한 경우, 최신 제출 1건만 공식 비교에 쓰는 것이 안정적임.

## Stage72 수정 내용

### 1. 켈리브레이션 표준편차 조회 범위 고정
- MOB: 동일 라운드 + 동일 참가자 + 동일 역할 카테고리(센서리/테크니컬)의 전체 일반 심사위원 최신 제출값 기준.
- IKRC: 동일 라운드 + 동일 컵/샘플의 전체 일반 센서리 심사위원 최신 제출값 기준.
- 로그인한 헤드 심사위원의 평가팀명은 표준편차 계산 행 집합을 제한하지 않도록 변경.

### 2. 중복 제출 최신값 처리
- 동일 심사위원/동일 역할/동일 팀/동일 모드의 중복 제출은 최신 제출 1건만 켈리브레이션 표준편차 계산에 반영.
- MOB/IKRC 모두 적용.

### 3. 헤드 기준 비교값 통일
- 여러 헤드 기준값이 존재할 경우, 화면의 `헤드차`는 현재 로그인한 헤드 개인값이 아니라 **전체 헤드 기준값 평균**과 비교.
- 따라서 같은 참가자/컵을 보면 어떤 헤드가 로그인해도 표준편차와 헤드차 기준이 동일하게 표시됨.

### 4. 기존 검수완료 상태 호환
- Stage71까지 저장된 팀별 검수완료 토큰도 읽을 수 있도록 legacy token fallback 유지.
- Stage72 이후 새로 저장되는 켈리브레이션 확인은 팀명 독립 토큰으로 저장.

## 유지된 기준
- 표준편차/헤드차 판정:
  - 0.00 ~ 1.00점: 양호
  - 1.01 ~ 2.00점: 주의
  - 2.00점 초과: 점검
- 헤드 기준 평가는 순위/총점 집계에서 제외.
- 일반 심사위원 최신 제출값만 켈리브레이션 표준편차 계산에 사용.

## 검수 결과
- `functions/api/rpc.js` 문법 검사 통과.
- `functions/api/health.js` 문법 검사 통과.
- assessment/admin/debriefing/registry/camera 화면 인라인 스크립트 문법 검사 통과.
- ZIP 무결성 검사 통과.

## 배포 후 확인 권장 시나리오
1. MOB 센서리 헤드 A 로그인 → 같은 참가자 켈리브레이션 표준편차 확인.
2. MOB 센서리 헤드 B 로그인 → 같은 참가자 켈리브레이션 표준편차가 동일한지 확인.
3. MOB 테크니컬 헤드도 동일하게 확인.
4. IKRC 헤드 A/B 로그인 → 같은 컵의 표준편차와 헤드차 기준이 동일한지 확인.
5. 동일 심사위원이 재제출한 경우 최신 제출값 1건만 반영되는지 확인.
