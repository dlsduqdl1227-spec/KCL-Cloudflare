# Stage78 라운드별 백업 탭 및 검수 화면 구분 강화

## 반영 목적
- 백업 엑셀에서 예선/본선/결선 탭 구분을 더 명확하게 했습니다.
- 검수 화면에서도 실제평가와 켈리브레이션을 시각적으로 구분해 운영 중 오판 가능성을 줄였습니다.

## 변경 내용
- 백업 엑셀 행에 `라운드구분` 컬럼을 추가했습니다.
  - `Preliminary`, `Qualifying`, `예선`은 `예선`으로 정리합니다.
  - `Main`, `본선`은 `본선`으로 정리합니다.
  - `Final`, `결선`은 `결선`으로 정리합니다.
- 백업 엑셀 시트명을 라운드 우선으로 변경했습니다.
  - `예선_실제평가`
  - `예선_켈리브레이션`
  - `본선_실제평가`
  - `본선_켈리브레이션`
  - `결선_실제평가`
  - `결선_켈리브레이션`
- 전체 백업 시트는 `전체_원본백업`, 전체 분리 시트는 `실제평가_전체`, `켈리브레이션_전체`로 표시합니다.
- 검수 목록 카드에 `실제평가`/`켈리브레이션` 배지를 추가했습니다.
- 검수 목록 통계에 `실제평가 n / 켈리브레이션 n`을 표시합니다.
- 켈리브레이션 카드의 상태 버튼을 일반 `검수완료` 토글로 바꾸지 못하게 비활성화했습니다.
- 검수 상세 화면 제목 아래에도 `구분: 실제평가` 또는 `구분: 켈리브레이션`을 표시합니다.

## 검증 결과
- `node --check functions/api/rpc.js` 통과.
- `public/assessment/index.html` inline script 문법 검사 통과.
- 로컬 Cloudflare Pages Functions `/api/health` 정상 응답 확인.
- KCR 테스트 데이터로 예선/결선, 실제평가/켈리브레이션 분리 확인.
- 백업 워크북 시트 생성 확인:
  - `안내`
  - `전체_원본백업`
  - `실제평가_전체`
  - `켈리브레이션_전체`
  - `예선_실제평가`
  - `예선_켈리브레이션`
  - `결선_실제평가`
  - `결선_켈리브레이션`
- 브라우저 검수 화면에서 실제평가/켈리브레이션 카드 배지와 켈리브레이션 상태 버튼 비활성화 확인.
- KCR, KCAC, MOC, MOB, KTCC, KBC, IKRC 전체 대회 코드에서 백업 API 정상 응답 확인.

## 유지 사항
- 선수등록용/운영계정등록용 엑셀 import 방식은 변경하지 않았습니다.
- 최종정리 엑셀은 기존처럼 검수완료/수정완료 및 순위 반영 대상 데이터만 포함합니다.
