## 2. 발견된 핵심 문제와 수정 내용

### 2.1 대회별 라운드 구조 불일치

문제:
- MOB는 규정집 기준 예선/결선 구조인데 관리자 화면에서 예선/본선/결선을 선택할 수 있었다.
- KCAC, KCR, IKRC, MOB처럼 본선이 없는 대회에서 과거 설정값이나 입력값이 `본선`으로 들어오면 서버 저장/순위/동점 규칙이 라운드를 다르게 해석할 수 있었다.

수정:
- MOB 관리자 라운드 옵션을 `예선`, `결선`으로 제한했다.
- 서버 공통 함수 `normalizeCompetitionRound()`를 추가해 대회별 라운드 구조를 통일했다.
- KCAC/MOB/KCR/IKRC에서 `본선` 값이 들어오면 운영상 `결선`으로 정규화되도록 처리했다.
- MOC/KBC는 규정집대로 `예선`, `본선`, `결선`을 유지한다.

### 2.2 본선/결선 시간 기준 오판 가능성

문제:
- KBC, MOC, MOB의 시간 제한은 예선과 본선/결선이 다르다.
- 서버 시간 계산 경로가 실제 한글 라운드명 `본선`, `결선`을 안정적으로 처리하지 못하면 본선/결선을 예선 기준으로 계산할 위험이 있었다.

수정:
- KBC: 예선 7분, 본/결선 10분 기준이 실제 한글 라운드명에서도 적용되도록 보강했다.
- MOC: 예선 5분, 본/결선 6분 기준과 본/결선 가산점 최대 3점 처리가 실제 한글 라운드명에서도 적용되도록 보강했다.
- MOB: 예선 10분, 결선 15분 기준이 실제 한글 라운드명에서도 적용되도록 보강했다.

### 2.3 참가자 배정 번호가 예선 컵번호로 고정되는 문제

문제:
- 참가자 호출 API가 현재 라운드와 관계없이 `prelim_cup_no`를 우선 사용했다.
- 본선/결선 진행 중에도 예선 컵번호가 화면에 잡힐 수 있어, 심사위원이 잘못된 참가자를 선택할 위험이 있었다.

수정:
- 현재 라운드가 예선이면 `prelim_cup_no`, 본선이면 `main_cup_no`, 결선이면 `final_cup_no`를 우선 사용하게 했다.
- KTCC는 규정 운영 방식에 맞게 팀번호 기준을 유지했다.
- 참가자 목록 정렬도 숫자형/문자형을 같이 고려하도록 보강했다.

### 2.4 점수 저장 라운드 불일치

문제:
- 화면은 KCAC/MOB 등을 결선처럼 계산하더라도 서버 저장은 원래 `current_round` 문자열을 그대로 넣었다.
- 같은 결선 점수가 `본선`과 `결선`으로 나뉘어 저장되면 검수/순위 조회가 비어 보이거나 분리될 수 있었다.

수정:
- `submitScores()` 저장 시 대회별 정규화 라운드를 사용하도록 변경했다.
- 반환값의 `round`도 실제 저장 라운드와 동일하게 맞췄다.

### 2.5 순위 집계가 라운드를 섞고 MAX 점수만 보던 문제

문제:
- 기존 순위 API는 모든 라운드를 한 번에 섞어 조회하고, 참가자별 `MAX(final_score)`만 사용했다.
- KBC/IKRC처럼 평균이 필요한 대회, KCAC/MOB처럼 여러 잔/역할 점수를 합산해야 하는 대회에서 운영 순위가 왜곡될 수 있었다.
- 같은 심사위원이 재제출한 경우 이전 제출값도 순위에 남을 수 있었다.

수정:
- 현재 라운드만 조회하도록 변경했다.
- 같은 참가자/심사위원/역할의 재제출은 최신 제출값 기준으로 묶는다.
- KBC, IKRC, KCR은 현재 라운드 평균 기준으로 집계한다.
- KCAC, MOB는 현재 라운드 합산 기준으로 집계한다.
- MOC, KTCC는 현재 라운드 최신 제출 기준으로 집계한다.
- MOB/IKRC 헤드 켈리브레이션 및 켈리브레이션 모드 제출은 순위에서 제외한다.

### 2.6 정상 제출이 실격으로 저장될 수 있던 문제

문제:
- `disqualified: 'N'` 같은 문자열은 JavaScript에서 truthy 값이므로 정상 제출이 서버에서 실격으로 저장될 수 있었다.

수정:
- `Y`, `DQ`, `실격`, `true`, `1`만 실격으로 처리하고, `N`, `정상`, `false`, `0`은 정상으로 처리하도록 보강했다.

### 2.7 MOB 시간감점이 총점으로 저장될 수 있던 문제

문제:
- MOB 제출 배열 뒤쪽에 시간감점과 경기시간이 붙어 있어, 서버가 뒤에서 처음 발견한 숫자를 총점으로 오인할 수 있었다.
- 추가 안정성 재검수에서 `extraFields['총점']`이 시간감점 반영 전 값으로 남을 수 있는 지점도 확인했다.

수정:
- 프론트에서 MOB `extraFields['총점']`을 명시적으로 보낸다.
- 시간감점 반영 후 최종 total을 다시 `extraFields['총점']`에 반영하도록 수정했다.
- 서버는 `최종점수`, `총점` 같은 명시 필드를 우선 사용하고, 대회별 총점 위치를 먼저 확인한 뒤에만 후방 숫자 스캔을 수행한다.

### 2.8 동점 기준 복원 안정성 보강

문제:
- 순위 API는 표준 제출 테이블의 `raw_json`을 기준으로 동점 기준을 복원한다.
- IKRC와 MOB는 원본 배열 안의 평가 항목명이 없으면 Flavor/Sweetness/Mouthfeel, Sensory/Technical/Creative 동점 기준이 누락될 수 있었다.

수정:
- IKRC 원본 제출 배열의 평가 항목 헤더를 서버에서 복원하도록 추가했다.
- MOB 원본 제출 배열의 테크니컬, 센서리, 창작음료 항목 헤더를 서버에서 복원하도록 추가했다.
- KBC/MOC/KTCC는 제출 시 `extraFields`에 동점 기준 값과 종료시간이 명시되어 있으므로 기존 구조를 유지했다.

### 2.9 KTCC 과거 라운드값 방어

문제:
- KTCC는 규정상 예선/결선 구조인데, DB나 과거 설정값에 `본선`이 들어 있으면 관리자 라운드 옵션에 다시 노출될 수 있었다.

수정:
- KTCC의 `본선`, `main`, 토너먼트 단계 표기는 모두 `결선`으로 정규화했다.

## 5. 배포 전 확인 사항

GitHub와 Cloudflare 배포 전 확인할 항목:
- 관리자 콘솔에서 MOB 라운드 옵션이 `예선`, `결선`만 보이는지 확인한다.
- 본선/결선 컵번호가 등록된 대회에서 참가자 선택 목록이 현재 라운드 컵번호로 뜨는지 확인한다.
- Cloudflare D1 원격 DB에 최신 스키마가 이미 반영되어 있는지 확인한다. 이번 수정은 새 컬럼을 추가하지 않는다.
- 운영 중 기존에 `MOB 본선`으로 저장된 제출 데이터가 있다면, 새 제출부터는 `결선`으로 저장된다. 과거 데이터 순위까지 합쳐야 하면 DB 데이터 정리가 별도로 필요하다.

## 6. 남은 운영 확인 항목

코드 문제가 아니라 운영 산식 확인이 필요한 항목:
- KCR 결선의 팀 평균/분포 기반 정규화 산식은 규정집에 내부 산식으로 표현되어 있어, 공식 운영 산식이 확정되면 별도 반영이 필요하다.
- KTCC 결선 마지막 라운드의 히든 미션은 현재 범용 정답수/가산점 구조에는 들어갈 수 있으나, 미션별 별도 입력 UI가 필요하면 추가 작업이 필요하다.

위 두 항목을 제외한 예선/본선/결선 라운드 구조, 시간 기준, 정상/실격 저장, 참가자 번호 호출, 기본 순위 집계 문제는 이번 업데이트에 반영했다.
