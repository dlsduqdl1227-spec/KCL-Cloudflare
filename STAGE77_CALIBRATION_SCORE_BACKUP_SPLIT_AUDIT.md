# Stage77 켈리브레이션/실제 대회평가 백업 분리 검수

## 반영 목적
- 점수·검수 백업 엑셀에서 켈리브레이션 데이터와 실제 대회 평가 데이터를 명확히 분리했습니다.
- 켈리브레이션 점수가 순위·공식 점수 자료로 오인되지 않도록 행 구분과 시트를 동시에 추가했습니다.

## 변경 기준
- `모드`가 켈리브레이션/Calibration이면 `켈리브레이션`으로 분류합니다.
- `검수상태`가 켈리브레이션이면 `켈리브레이션`으로 분류합니다.
- MOB/IKRC의 헤드 심사위원 입력은 실제 순위 반영 점수가 아니므로 `켈리브레이션`으로 분류합니다.
- 실제 평가 행도 미검수 상태이면 `순위반영여부`는 `N`, 검수완료/수정완료 후에는 기존 순위 로직에 따라 `Y`가 됩니다.

## 변경 파일
- `functions/api/rpc.js`
  - 백업 행에 `백업구분`, `순위반영여부`, `순위반영제외사유` 컬럼 추가.
  - 백업 응답에 `competitionRows`, `calibrationRows`, `categoryCounts` 추가.
- `public/assessment/index.html`
  - 점수·검수 백업 엑셀 시트를 다음처럼 분리:
    - `전체_점수검수`
    - `실제대회평가_전체`
    - `켈리브레이션_전체`
    - 라운드별 `평가_예선/본선/결선`
    - 라운드별 `켈리브레이션_예선/본선/결선`

## 검증 결과
- `node --check functions/api/rpc.js` 통과.
- `public/assessment/index.html` inline script 문법 검사 통과.
- 로컬 Cloudflare Pages Functions `/api/health` 정상 응답 확인.
- KCR 실제 평가 1건과 켈리브레이션 1건 제출 후 백업 API 확인:
  - 실제 평가: `백업구분=실제대회평가`
  - 켈리브레이션: `백업구분=켈리브레이션`, `순위반영여부=N`
  - 실제 평가 미검수: `순위반영여부=N`
  - 실제 평가 검수완료 후: `순위반영여부=Y`
- KCR, KCAC, MOC, MOB, KTCC, KBC, IKRC 전체 대회 코드에서 백업 API 정상 응답 확인.
- Chrome 브라우저에서 워크북 생성 시트 확인:
  - `안내`
  - `전체_점수검수`
  - `실제대회평가_전체`
  - `켈리브레이션_전체`
  - `평가_예선`
  - `켈리브레이션_예선`

## 유지 사항
- 선수등록용/운영계정등록용 엑셀 import 방식은 변경하지 않았습니다.
- 최종정리 엑셀은 기존처럼 검수완료/수정완료이면서 순위 반영 대상인 데이터만 포함합니다.
