# KCL Stage73 KCAC 미디어 입력 분리 및 앨범 불러오기 업데이트

## 목적
KCAC 평가 화면에서 기존 `사진 선택 / 촬영` 버튼이 하나로 묶여 있어 모바일 브라우저에서 촬영 모드와 앨범 선택 모드가 혼재되는 문제를 수정했다. 특히 사진을 촬영한 뒤 실제 이미지가 안정적으로 로드되지 않고 가이드선만 보이는 상황을 줄이기 위해 촬영 입력과 앨범 입력을 완전히 분리했다.

## 수정 범위
- `public/assessment/index.html`
- `public/camera/index.html`

## 적용 내용

### 1. KCAC 평가 화면 입력 분리
기존 단일 버튼:
- `사진 선택 / 촬영`

변경 후:
- `촬영하기`
- `앨범에서 사진 불러오기`
- `앨범에서 영상 불러오기`

각 버튼은 서로 다른 input을 사용한다.

| 기능 | input 설정 | 비고 |
|---|---|---|
| 촬영하기 | `accept="image/*" capture="environment"` | 카메라 촬영 전용 |
| 앨범에서 사진 불러오기 | `accept="image/*"` | 앨범 사진 선택 전용. capture 없음 |
| 앨범에서 영상 불러오기 | `accept="video/*"` | 앨범 영상 선택 전용 |

### 2. 영상 불러오기 방식
영상 원본은 D1/브라우저 저장 용량 문제를 막기 위해 그대로 저장하지 않는다.
선택한 영상에서 첫 평가용 프레임을 추출해 이미지로 변환한 뒤, 기존 가이드선 편집 및 스냅샷 저장 흐름을 사용한다.

저장되는 정보:
- 추출된 프레임 이미지
- 미디어 입력 방식
- 파일명
- 영상 길이(초)
- 스냅샷 이미지

원본 영상 파일 자체는 저장하지 않는다.

### 3. 사용자 안내 문구 보강
KCAC 평가 화면에 현재 불러온 자료가 다음 중 무엇인지 표시하도록 했다.
- 촬영 사진
- 앨범 사진
- 앨범 영상 프레임

안내 문구도 `사진` 단일 기준에서 `사진/영상` 기준으로 수정했다.

### 4. 저장/제출 흐름 유지
KCAC 제출 검증은 기존처럼 평가 이미지 스냅샷 저장을 요구한다.
즉, 사진 또는 영상 프레임을 불러온 뒤 반드시 `가이드 함께 저장`을 눌러야 제출 기록에 남는다.

### 5. 독립 KCAC 컵 가이드 페이지도 동일 방향으로 보강
`/camera/` 페이지도 다음처럼 분리했다.
- 촬영하기
- 카메라 중지
- 앨범에서 사진 불러오기
- 앨범에서 영상 불러오기

## 배포 후 확인 권장
1. KCAC 예선 평가 화면 진입
2. `촬영하기` 버튼으로 카메라 촬영 후 이미지가 표시되는지 확인
3. `앨범에서 사진 불러오기` 버튼으로 기존 사진 선택 후 이미지가 표시되는지 확인
4. `앨범에서 영상 불러오기` 버튼으로 영상 선택 후 프레임 이미지가 표시되는지 확인
5. 가이드 위치 조정 후 `가이드 함께 저장` 클릭
6. 전체 제출 후 관리자 검수/디브리핑에서 이미지 스냅샷이 확인되는지 점검
