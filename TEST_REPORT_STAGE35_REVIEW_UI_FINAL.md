# TEST REPORT - STAGE35 REVIEW UI FINAL

## 목적
검수 상세 화면에서 기본 정보/관리용 메타 정보가 과도하게 노출되어 실제 평가 화면과 다르게 보이는 문제를 정리하고, 평가 항목·코멘트·실격/증빙 중심으로만 확인/수정할 수 있도록 검수 UI를 재구성했습니다.

## 반영 사항

1. 검수 상세 화면 간소화
- `기본 정보` 섹션 제거
- 제출시간, 대회코드, 라운드, 심사위원명, 역할, 모드 등 평가 수정과 직접 관련 없는 메타 정보는 상세 편집 영역에서 숨김
- 상단 제목/부제목과 목록 화면에서 필요한 식별 정보는 확인 가능하도록 유지

2. 실제 평가 화면 중심 구성
- 점수 항목, 강도 항목, 스마트태그, 코멘트, O/X, 종료시간, 실격 여부/사유, 저장 이미지/서명만 표시
- KCR/IKRC/KBC 등 자동 총점 재계산에 필요한 hidden input은 UI 노출 없이 유지
- MOC/KTCC는 정답수·가산점·종료시간 중심으로 표시

3. 안내/겹침 정리
- 검수 상세 상단 안내 박스 숨김
- 자동저장 안내 영역을 고정(sticky)에서 일반 영역으로 변경하여 평가 항목을 덮지 않도록 수정
- `수정완료/검수완료` 흐름은 유지

4. 전체 코멘트 처리 유지
- 항목별 코멘트가 없고 전체 코멘트만 있는 경우도 검수 화면과 PDF/리포트에 반영되도록 유지

## 대회별 총점 재계산 확인

- KCR: Flavor 수정 후 Sweetness x2 포함 총점 재계산 확인
- MOC: 정답수 + 가산점 수정 후 총점 재계산 확인
- KTCC: Section1/2/3 + 가산점 수정 후 총점 재계산 확인
- IKRC: 가중치 항목 수정 후 총점 재계산 확인
- KBC: 가중치 및 시간감점 반영 재계산 확인
- MOB: 센서리/테크니컬/창작 항목 수정 후 재계산 확인
- KCAC: 최종점수 수정 후 총점 재계산 확인

## 운영 기준

Stage35는 Stage34의 점수 재계산 안정화 기준을 유지하면서, 검수 상세 화면을 실제 평가 항목 중심으로 정리한 버전입니다.
