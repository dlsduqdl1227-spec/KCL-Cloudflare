# KCL Stage68 MOB 테크니컬 시간감점 / 순위 집계 수정 기록

## 확인한 문제

MOB 테스트 중 테크니컬 심사 데이터에서 경기 시간이 10분 41초로 입력된 경우, 규정상 실격이 아니라 31~60초 초과 구간의 40점 감점 대상입니다. 기존 화면은 이 감점이 적용된 결과를 중간 계산 없이 0점으로만 보여주어, 감점 처리인지 오류인지 확인하기 어려웠습니다.

또한 순위 상세 디브리핑에 MOB 헤드/켈리브레이션 성격의 데이터가 같이 표시될 수 있어, 실제 순위 반영 데이터와 상세 화면 데이터가 섞여 보이는 문제가 있었습니다.

## 수정 내용

1. MOB 테크니컬 총점 키 호환 보강
   - `테크니컬 총점`뿐 아니라 `테크니컬총점`도 공식 계산에 인식하도록 보강했습니다.
   - `센서리총점`, `창작메뉴총점`도 동일하게 보강했습니다.

2. 시간감점 표시 구조 보강
   - `감점 전 합산`
   - `시간감점`
   - `감점 적용 후 점수`
   - `순위 반영점수`
   을 분리했습니다.

3. MOB 순위 집계 방식 명확화
   - 센서리 평균 + 테크니컬 점수 + 창작메뉴 평균 - 시간감점 방식으로 계산합니다.
   - 계산 결과가 음수일 경우 최종 순위 반영 점수는 0점으로 표시합니다.
   - 단, 중간값인 `감점 적용 후 점수`는 음수 그대로 보여 계산 근거를 확인할 수 있게 했습니다.

4. MOB 헤드/켈리브레이션 데이터 제외 보강
   - 순위 집계뿐 아니라 순위 상세 디브리핑과 최종정리 엑셀에서도 헤드/켈리브레이션 성격의 데이터가 섞이지 않도록 했습니다.
   - `역할`, `role`뿐 아니라 `심사위원역할`, `judgeRole`, `Judge Role`도 헤드 판정에 사용하도록 보강했습니다.

5. 순위 상세 화면 보강
   - MOB 상세 디브리핑 상단에 계산 요약 박스를 추가했습니다.
   - 예: 센서리 평균, 테크니컬 점수, 감점 전 합산, 시간감점, 감점 적용 후, 최종 순위 반영 점수를 한 번에 확인할 수 있습니다.

## 테스트 결과

코드 기준 테스트를 진행했습니다.

- `functions/api/rpc.js` 문법 검사 통과
- `public/assessment/index.html` 내부 스크립트 문법 검사 통과
- MOB 테크니컬 총점 14.8점 + 시간감점 40점 → 감점 적용 후 -25.2점, 순위 반영 0점 확인
- MOB 홍길동 샘플 계산 확인
  - 센서리 평균: 20.8667
  - 테크니컬: 14.8
  - 감점 전 합산: 35.667
  - 시간감점: 40
  - 감점 적용 후: -4.333
  - 최종 순위 반영: 0
- `10분 41초` → 641초 파싱 확인
- MOB 헤드 데이터가 순위 반영 데이터에서 제외되는지 확인

## 아직 별도 확인이 필요한 범위

실제 운영 DB에서 제출 → 검수완료/수정완료 → 실시간 순위 → 디브리핑 → 최종정리 엑셀 다운로드까지의 브라우저 기반 전체 클릭 테스트는 별도로 진행해야 합니다.
