# KCL Assessment Stage13 Test Report

## 반영 항목

1. 등록·권한 관리 화면 개편
- `/registry/` 화면을 `선수 / 심사위원 & 팀장 / 관리자` 탭 구조로 분리했습니다.
- 선수 탭: 선수·팀 일괄 등록, 개별 등록, 등록 목록, 선수 초기화.
- 심사위원 & 팀장 탭: 심사위원·팀장·운영진 일괄 등록, 개별 추가, 권한 목록, 운영 초기화.
- 관리자 탭: 관리자 이름/핸드폰번호 개별 추가, 관리자 목록 확인, 관리자 삭제.

2. 선수 초기화 버튼
- 선택 대회의 선수 등록 데이터만 삭제합니다.
- 점수 데이터는 삭제하지 않습니다.
- 버튼 클릭 후 confirm 확인 + 지정 문구 입력 확인을 거쳐야 실행됩니다.
- 입력 문구 예: `KCR 선수초기화`

3. 운영 초기화 버튼
- 선택 대회의 심사위원·팀장·운영진 권한만 초기화합니다.
- 관리자 계정은 삭제 대상에서 제외했습니다.
- 여러 대회 권한을 가진 계정은 선택 대회 권한만 제거하고, 남은 대회 권한은 유지합니다.
- 버튼 클릭 후 confirm 확인 + 지정 문구 입력 확인을 거쳐야 실행됩니다.
- 입력 문구 예: `KCR 운영초기화`

4. 관리자 추가 기능
- 관리자 탭에서 이름/핸드폰번호/소속을 입력해 관리자 계정을 추가할 수 있습니다.
- 서버는 `ADMIN` 계정을 저장할 때 권한을 자동으로 `ALL`, 역할을 `관리자`로 고정합니다.
- 관리자 홈 로그인 기준:
  - 이름 = 관리자 아이디
  - 핸드폰번호 = 관리자 비밀번호
  - 시크릿 코드 = 기존 `5061` 또는 환경변수 값

5. 안전장치
- 운영 초기화는 관리자 계정을 삭제하지 않습니다.
- 마지막 관리자 계정은 삭제할 수 없도록 서버에서 차단합니다.
- 권한 삭제/초기화는 전체 관리자 권한이 있어야 실행됩니다.

## 점검 결과

- `functions/api/rpc.js` Node syntax check 통과.
- `/registry/` 내 모든 inline script syntax check 통과.
- `/admin/`, `/assessment/`, `/debriefing/`, `/camera/` inline script syntax check 통과.
- 새 RPC 액션 확인:
  - `listOperators`
  - `clearOperators`
- 기존 RPC 액션 유지:
  - `upsertOperatorAccount`
  - `deleteOperatorAccount`
  - `clearParticipants`
  - `importParticipants`
  - `importOperators`

## 업데이트 후 예상 문제 및 방지 조치

1. 관리자 계정을 실수로 삭제해 접속이 끊기는 문제
- 방지: 마지막 관리자 삭제 차단.

2. 운영 초기화 시 관리자까지 삭제되는 문제
- 방지: `ADMIN`, `ALL`, 관리자 역할 계정은 운영 초기화 대상에서 제외.

3. 여러 대회 권한을 가진 심사위원/팀장의 권한이 통째로 삭제되는 문제
- 방지: 선택 대회 코드만 제거하고, 남은 대회 권한은 유지.

4. 버튼 오클릭으로 대량 삭제되는 문제
- 방지: confirm 1회 + 정확한 문구 입력 1회로 이중 확인.

5. 기존 엑셀 업로드 방식과 충돌
- 방지: 기존 CSV/엑셀 일괄 업로드 함수는 유지했고, 개별 관리자 추가 기능만 별도 탭에 추가.

## 최종 판단

Stage13 파일은 기존 Stage12 기능을 유지하면서 등록·권한 관리 화면의 구조를 개선하고, 선수/운영 초기화 및 관리자 추가 기능을 반영한 버전입니다.
