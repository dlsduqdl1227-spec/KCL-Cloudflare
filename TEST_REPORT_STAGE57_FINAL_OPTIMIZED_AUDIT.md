# TEST REPORT — Stage57 Final Optimized Audit

## 목적
Stage56 MOB 점수집계 보정본을 기준으로 전체 대회 구조, 점수 재계산, 실격 처리, 켈리브레이션 진입 방식, 중복 UI 숨김 처리, 배포 파일 무결성을 다시 점검하고 최종 안정화본으로 패키징했다.

## 최종 반영 사항

### 1. KCR 켈리브레이션 방식 유지 확인
- KCR은 관리자 화면의 별도 버튼 방식이 아니라 기존처럼 KCR 평가 설정 화면 안의 `켈리브레이션 모드` 체크박스로 선택한다.
- KCR 켈리브레이션 제출값은 모드에 `켈리브레이션`이 포함되어 저장되며, 순위 집계에서는 제외된다.

### 2. MOB 점수집계 Stage56 보정 유지
- `10분 08.00초` → 608초로 정상 파싱한다.
- `10:08` → 608초로 정상 파싱한다.
- 시간감점이 `-6`처럼 음수로 저장되어 있어도 6점 감점으로 정규화한다.
- 실격 그룹은 순위 집계에서 0점 및 실격으로 처리한다.
- 테크니컬은 `테크니컬 총점 / 시간감점 / 총평가 반영점수` 분리 구조를 유지한다.
- 센서리는 중복 합산 표시 없이 `센서리 반영점수` 중심 표시를 유지한다.

### 3. 전체 대회 실격 점수 보존 보강
- KCR / KBC / KCAC / MOB / IKRC / MOC / KTCC 공통으로 `실격여부=Y`, `실격`, `true`, `1` 등이 들어오면 서버 재계산 점수를 0점으로 보존하도록 보강했다.
- 기존에는 일부 대회에서 실격 상태임에도 원시 평가항목이 남아 있으면 서버 재계산 또는 순위 상세에서 점수가 다시 계산될 수 있는 가능성이 있었다.

### 4. KBC 시간감점 보강
- KBC 서버 재계산에서 과거 저장값 또는 검수 수정값이 `-6`처럼 음수 감점으로 들어와도 점수가 증가하지 않도록 `abs()` 기반 양수 감점으로 정규화했다.

### 5. KCAC 감점 보강
- KCAC 리프수감점, 시간감점, 일반 감점도 음수 저장값을 양수 감점으로 정규화했다.
- 단, 실격성 감점으로 쓰는 999 이상 값은 실격/0점 처리 의미를 유지한다.

### 6. 중복 UI 정리 유지
- MOB 하단 중복 점수 박스 숨김 유지
- MOB 코멘트 참고 영역 숨김 유지
- KCR / KBC / KCAC / IKRC 검수 화면의 공식 항목처럼 오해될 수 있는 참고 요약 영역 숨김 유지
- 코멘트 자동 생성 및 저장 기능은 유지
